# Phase 6B - Discovery to Contract Bridge

## Purpose

Convert RAKSO discovery notes into a clean SDD+ contract draft.

This is not product work. It is not app planning. It is not engineering work.

## Main Output

Phase 6B produces one practical packet with five sections:

1. Evidence Summary
2. Pain Map
3. Money Logic
4. Offer Direction
5. SDD+ Contract Draft

## Hard Boundary

Do not create or plan:

- software
- app screens
- database structure
- automations
- agents
- integrations
- deployment
- test systems
- runtime systems

Phase 6B is only for turning discovery into a decision-ready contract draft.

## Input Material

Use existing RAKSO discovery only:

- TaxPak Discovery
- Bookkeeping AI Discovery
- RAKSO notes
- SDD+ decisions already accepted by the user

No invented evidence.
No market claims without source.
No assumptions disguised as facts.

## Flow

Raw discovery becomes:

- evidence-backed statements
- grouped operational pains
- clear business value
- non-technical offer direction
- SDD+ contract draft

## Section 1 - Evidence Summary

Capture:

- evidence item
- source or origin
- user affected
- problem observed
- why it matters
- confidence level

## Section 2 - Pain Map

Capture:

- pain area
- symptom
- root cause
- current workaround
- cost of doing nothing
- urgency

## Section 3 - Money Logic

Capture:

- time saved
- errors reduced
- revenue unlocked
- client capacity increased
- why the buyer would care now

## Section 4 - Offer Direction

Capture:

- target user
- job to be done
- core promise
- before state
- after state
- MVP boundary
- what is not included

## Section 5 - SDD+ Contract Draft

Capture:

- contract name
- problem
- target user
- allowed scope
- forbidden scope
- inputs
- outputs
- acceptance criteria
- exit criteria
- human approval requirement

## First Use Case

Start with TaxPak Discovery.

Reason: TaxPak is the first RAKSO validation and cashflow vehicle. It has the clearest discovery boundary and should become the first contract draft.

## Second Use Case

Bookkeeping AI Discovery comes after TaxPak.

Reason: Bookkeeping is adjacent and useful, but it should not hijack the first contract sequence.

## Rule

One discovery project at a time.

Do not mix TaxPak and Bookkeeping into the same first draft.

## Immediate Next Step

Compile TaxPak Discovery into the five Phase 6B sections.
