# AGENTS.md

## Repository Mission

This repository defines the RAKSO Solution Compiler framework.

It is a repo-native, tool-agnostic, model-agnostic framework for converting business chaos into evidence, diagnosis, money logic, offer direction, SDD+ contracts, and controlled execution scopes when authorized.

This repository is not a vehicle repository and must not contain vehicle-specific discovery, product scope, MVP definition, build plans, or implementation details.

## Core Formula

```txt
Reality -> Evidence -> Diagnosis -> Money -> Offer -> Contract -> Authorized Execution
```

## Capability Roles

RAKSO is defined by capabilities, not vendor names.

| Capability | Role |
|---|---|
| Command Brain | Strategy, synthesis, blindspots, adversarial thinking, decision framing |
| Primary Executor | Authorized repo work when a committed contract allows it |
| Secondary Executor | Supporting documentation, tooling, cleanup, or auxiliary work when authorized |
| Orchestration Layer | Multi-worker coordination only when authorized |
| External Audit Layer | Independent review, red-team logic, validation, and second opinion |
| Rules Engine | SDD+ contracts, lifecycle rules, allowed scope, forbidden scope |
| Source of Truth | Versioned repo, committed artifacts, state, logs, audit trail |
| Drift Guard | Lightweight checks for state, stale wording, forbidden surface, and next action |
| Human Operator | Final authority, approval, denial, redirect, override, stop |

Examples may include ChatGPT, Codex, Jules, Antigravity, Gemini, DeepSeek, Claude, Cursor, OpenHands, LangGraph, CrewAI Flows, Temporal, Conductor, GitHub, GitLab, local Git, local LLMs, and human operators.

Examples are replaceable. Capabilities are the stable contract.

## Operating Rules For Agents / LLMs / Human Executors

1. Do not start with technology.
2. Do not invent business pain.
3. Do not mix evidence with assumptions.
4. Do not recommend AI, agents, custom apps, workflows, or automation when a simpler solution captures value.
5. Do not create implementation artifacts before a committed SDD+ contract authorizes them.
6. Do not remove Critical Lens checkpoints.
7. Do not make the system dependent on a single model, vendor, coding assistant, orchestration platform, repo host, or runtime.
8. Prefer small, useful, committed artifacts over broad documents that do not change the operating state.
9. Respect Velocity Mode: decide target, produce artifact or decision, audit once, move on.
10. Keep vehicle-specific work outside this repository.

## Required Behavior For Repo Tasks

When modifying this repo:

1. State what layer is affected.
2. State whether the change is doctrine, framework, lifecycle, template, guard, example, or authorized execution.
3. Preserve the core formula.
4. Flag assumptions explicitly.
5. Avoid repeated permission loops when a committed contract already authorizes the action.
6. Verify the delta after changes.
7. Do not add vehicle-specific discovery or product scope to this repo.

## Forbidden Shortcuts

- Transcript -> solution without evidence ledger
- Pain -> app backlog without pain-to-profit
- Offer -> implementation without SDD+ contract
- Lead research -> truth claims without validation
- Agent execution -> production writes without approval policy
- Vehicle discovery -> compiler repo contamination
- Vendor preference -> permanent dependency

## Current Forbidden Surface

Unless explicitly authorized by a later committed SDD+ contract, do not create or modify:

- product implementation code
- product schemas
- executable modules
- runtime integrations
- automation workflows
- product artifacts
- prompts, tools, agents, or AI system behavior
- runtime orchestration
- executable interfaces or APIs
- data models
- infrastructure or deployment behavior
- workflow automation
- product build authorization
- vehicle-specific discovery or product scope

## Velocity Rule

Manual and clear first. Automated later.

When the target is clear and authorized, execute inside the contract.

When authorization is unclear, stop and escalate.
